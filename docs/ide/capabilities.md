# 주요 기능 상세 (Core Capabilities)

CodePilot IDE가 제공하는 강력한 개발 보조 기능들을 상세하게 소개합니다.  
단순한 채팅을 넘어, 실제 코드 작성과 수정, 실행까지 개발자의 손발이 되어주는 기능들입니다.

---

## 1. 멀티 LLM 지원 및 하이브리드 운영

작업의 목적과 보안 요구사항에 따라 최적의 AI 모델을 유연하게 선택하여 사용할 수 있습니다.

### 🔹 기능 상세
*   **다양한 모델 지원**: GPT-4, Claude 3.5 Sonnet, Haiku 등 최신 고성능 모델뿐만 아니라, Ollama 기반의 로컬 모델도 지원합니다.
*   **모델 라우팅**:
    *   **코딩 작업**: 논리적 추론이 필요한 작업은 고성능 모델(Claude 3.5 Sonnet 등)이 처리합니다.
    *   **단순 질의**: 빠른 응답이 필요한 질문은 경량 모델(Haiku, GPT-3.5)이 처리하여 비용을 절감합니다.
*   **보안 모드 (Local Only)**: 인터넷 연결이 제한되거나 보안이 중요한 프로젝트에서는 사내 서버 또는 로컬 PC에 설치된 LLM만 사용하도록 강제할 수 있습니다.

### 💡 활용 예시
> "평소에는 Claude 3.5로 개발하다가, 사내 핵심 보안 모듈을 다룰 때는 자동으로 사내 구축형 Ollama 모델로 전환되어 데이터 유출을 원천 차단합니다."

---

## 2. 인라인 Diff 미리보기 (Inline Diff)

AI가 제안한 코드를 무조건 반영하지 않고, 에디터 내에서 변경 전후를 직관적으로 비교하고 선택할 수 있습니다.

### 🔹 기능 상세
*   **실시간 비교**: 현재 열려있는 파일 위에 AI의 수정 제안이 오버레이(Overlay) 형태로 표시됩니다.
*   **부분 수락/거절**: 전체 코드를 다 바꿀 필요 없이, 마음에 드는 라인만 선택하여 반영(`Accept`)하거나 거절(`Reject`)할 수 있습니다.
*   **코드 렌즈(CodeLens) 통합**: 클릭 한 번으로 변경 사항을 적용하거나 취소할 수 있는 버튼을 제공합니다.

### 💡 활용 예시
> "AI가 리팩터링한 코드 중 변수명 변경은 마음에 들지만 로직 변경은 위험해 보일 때, 변수명 변경 부분만 클릭해서 반영하고 나머지는 거절할 수 있습니다."

## 3. 멀티파일 동시 편집 (Multi-file Editing)

하나의 요청으로 연관된 여러 파일을 동시에 분석하고 수정할 수 있습니다.

### 🔹 기능 상세
*   **의존성 분석**: 함수 시그니처를 변경하면, 해당 함수를 호출하는 다른 파일들의 코드도 함께 찾아 수정합니다.
*   **일괄 적용**: "User 모델에 필드 추가해줘"라고 요청하면, `User.java` (엔티티), `UserDto.java` (DTO), `UserRepository.java` (DB) 파일을 모두 찾아 동시에 수정 제안을 만듭니다.
*   **컨텍스트 유지**: 여러 파일 간의 문맥을 이해하고 있어, 파일 A의 변경이 파일 B에 미칠 영향을 고려하여 코드를 작성합니다.

### 💡 활용 예시
> "API 응답 포맷을 변경해야 할 때, 컨트롤러, 서비스, 테스트 코드까지 3개의 파일을 한 번에 수정 요청하여 작업 시간을 획기적으로 단축합니다."

## 4. 터미널 명령 자동 실행 (Auto Command Execution)

개발 환경 설정, 빌드, 테스트 실행 등 반복적인 터미널 작업을 AI가 대신 수행합니다.

### 🔹 기능 상세
*   **명령어 생성 및 실행**: 사용자의 의도를 파악하여 적절한 쉘 명령어(Shell Command)를 생성하고, 승인 시 터미널에서 즉시 실행합니다.
*   **OS 자동 감지**: Windows(PowerShell), macOS/Linux(Bash, Zsh) 등 사용자의 운영체제에 맞는 문법으로 명령어를 변환합니다.
*   **결과 분석**: 명령어 실행 결과를 AI가 읽고, 성공 여부를 판단하여 다음 작업을 진행하거나 에러 해결책을 제시합니다.

### 💡 활용 예시
> "'이 프로젝트 실행해줘'라고 말하면, `npm install`로 의존성을 설치하고 `npm start`로 서버를 띄우는 과정까지 알아서 진행합니다."

## 5. 자동 에러 수정 및 재시도 (Auto-Fix & Retry)

개발 중 발생하는 에러를 AI가 실시간으로 감지하고, 스스로 수정을 시도합니다.

### 🔹 기능 상세
*   **에러 감지**: 터미널의 실행 결과나 컴파일러의 에러 메시지를 모니터링하다가 실패가 감지되면 즉시 개입합니다.
*   **원인 분석 및 수정**: 에러 로그를 분석하여 원인을 파악하고, 코드를 수정하거나 실행 명령어를 고쳐서 다시 시도합니다.
*   **재시도 루프**: 설정된 횟수(예: 3회)만큼 수정을 반복하며 문제 해결을 시도하고, 해결되지 않으면 사용자에게 리포트합니다.

### 💡 활용 예시
> "테스트 코드를 돌렸는데 널 포인터 예외(NPE)가 발생하면, AI가 즉시 해당 라인을 찾아 `if (obj != null)` 체크 로직을 추가하고 테스트를 다시 돌려 통과시킵니다."

## 6. 자동화 워크플로우 (Orchestration)

복잡한 작업을 작은 단위로 쪼개어 순차적으로 혹은 병렬로 처리하는 에이전트 오케스트레이션 기능입니다.

### 🔹 기능 상세
*   **작업 분해 (Task Breakdown)**: "쇼핑몰 로그인 기능 만들어줘"라는 큰 요청을 받으면, 1) DB 스키마 설계, 2) API 구현, 3) 프론트엔드 UI 작성 등으로 작업을 나눕니다.
*   **순차 실행**: 앞 단계의 결과물(예: DB 설계)을 바탕으로 다음 단계(API 구현)를 진행하여 논리적 정합성을 유지합니다.
*   **도구 연동**: 필요에 따라 웹 검색, 파일 읽기, 터미널 실행 등 다양한 도구를 적재적소에 호출하여 작업을 완수합니다.

### 💡 활용 예시
> "새로운 기능을 구현할 때, 기획서 분석부터 파일 생성, 코드 작성, 테스트 실행까지의 과정을 AI가 설계하고 단계별로 실행하여 개발자를 리딩합니다."
