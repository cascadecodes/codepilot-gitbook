# IDE 주요 기능 (Core Features)

CodePilot IDE는 단순한 코딩 도구를 넘어, 개발자의 생산성을 극대화하는 강력한 엔진을 탑재하고 있습니다.  
사용자에게 보이는 **핵심 기능**과 그 뒤에서 작동하는 **내부 동작 원리**를 통합하여 소개합니다.

---

## 1. 대화형 개발 지원 (Chat & Coding)

개발자가 자연어로 요구사항을 말하면, AI가 의도를 파악하고 코드를 작성합니다.

* **요구사항 분석**: 모호한 요청을 구체화하여 작업 단계 제안
* **코드 생성**: 초안 코드부터 완성형 코드까지 즉시 생성
* **수정 및 개선**: 기존 코드를 분석하여 리팩터링 제안

{% hint style="info" %}
**고객 가치**

* 아이디어를 코드로 옮기는 **초기 진입 장벽 제거**
* 신입/주니어 개발자도 시니어급 가이드를 받으며 작업 가능
{% endhint %}

---

## 2. 지능형 모델 라우팅 (Smart Routing)

사용자의 모든 입력을 비싼 고성능 모델로 처리하지 않습니다. 입력의 의도를 파악하여 가장 적절한 모델과 도구로 연결합니다.

### ⚙️ 내부 동작 원리
1. **의도 분류기**: 사용자의 입력이 "단순 인사", "코드 생성", "파일 검색", "명령 실행" 중 무엇인지 아주 빠른 경량 모델이 먼저 판단합니다.
2. **모델 스위칭**: 
   * 복잡한 로직 설계 → `Claude 3.5 Sonnet` / `GPT-4`
   * 단순 채팅/요약 → `Haiku` / `GPT-3.5`
   * 보안 민감 데이터 → `Local Ollama`

{% hint style="info" %}
**고객 가치**

* **비용 절감**: 불필요하게 비싼 모델을 쓰는 낭비 방지
* **사용성 향상**: 모델 선택 고민 없이 작업에만 집중
{% endhint %}

---

## 3. 멀티파일 컨텍스트 인식 및 동시 수정 (Multi-file Edit)

CodePilot은 단일 파일만 보고 코드를 짜지 않습니다. 프로젝트 전체의 구조와 의존성을 이해하고, 한 번의 요청으로 여러 파일을 동시에 수정합니다.

### ⚙️ 내부 동작 원리
1. **의존성 그래프 분석**: 수정하려는 파일이 참조하고 있는 다른 파일(Import)이나, 반대로 해당 파일을 참조하는 곳(Usage)을 탐색합니다.
2. **변경 전파**: 함수 파라미터가 변경되면, 해당 함수를 호출하는 모든 파일의 호출부까지 찾아내어 수정 목록에 포함합니다.
3. **일괄 트랜잭션**: 여러 파일에 대한 수정 사항을 생성한 뒤, 사용자에게 한 번에 제안합니다.

{% hint style="info" %}
**고객 가치**

* **생산성 극대화**: 복잡한 리팩터링이나 기능 추가 작업을 빠르게 수행
* **오류 감소**: 파일 간 불일치로 인한 에러를 사전에 방지
{% endhint %}

## 4. 실시간 인라인 Diff 렌더링 (Inline Diff Rendering)

별도의 비교 창(Diff View)을 띄우지 않고, 현재 작업 중인 에디터 코드 줄 사이에 변경 사항을 자연스럽게 끼워 넣어 보여줍니다.

### ⚙️ 내부 동작 원리
1. **가상 렌더링**: 원본 코드를 건드리지 않고, VS Code의 `TextEditorDecorationType`을 활용하여 가상의 코드 라인을 렌더링합니다.
2. **스트리밍 Diff**: AI가 코드를 생성하는 즉시 실시간으로 Diff가 그려지므로, 완료될 때까지 기다릴 필요 없이 미리 검토할 수 있습니다.
3. **블록 단위 제어**: 변경 사항을 덩어리(Hunk) 단위로 관리하여, 사용자가 원하는 부분만 클릭 한 번으로 `Accept(반영)`하거나 `Reject(거절)`할 수 있습니다.

{% hint style="info" %}
**고객 가치**

* **코드 품질 유지**: 변경 사항을 꼼꼼히 검토하여 버그 발생 최소화
* **개발 속도 향상**: 번거로운 비교 과정 없이 즉시 확인 및 적용 가능
{% endhint %}

## 5. 터미널 세션 제어 및 자동 실행 (Terminal Automation)

CodePilot은 단순한 명령어 실행기가 아닙니다. 실제 개발자처럼 터미널 세션을 유지하고, 상황을 판단하며 명령어를 운용합니다.

### ⚙️ 내부 동작 원리
1. **쉘 세션 유지**: 일회성 명령 실행(`exec`)이 아니라, 지속적인 쉘 프로세스(`spawn`)를 관리하여 환경 변수나 작업 디렉토리(cwd) 상태를 유지합니다.
2. **출력 스트림 분석**: 터미널의 `stdout`, `stderr`를 실시간으로 훅(Hook)하여, 명령어가 성공했는지, 추가 입력이 필요한지, 에러가 났는지 판단합니다.
3. **OS/쉘 감지**: 사용자가 Windows(PowerShell)인지 macOS(Zsh)인지 감지하여, 동일한 요청이라도 OS에 맞는 문법으로 명령어를 변환해 실행합니다.

{% hint style="info" %}
**고객 가치**

* **편의성 증대**: 복잡한 명령어를 외우거나 타이핑할 필요 없음
* **문제 해결 속도 향상**: 실행 결과에 대한 즉각적인 피드백 및 조치 가능
{% endhint %}

---

## 6. 에러 패턴 분석 및 자가 치유 (Self-Healing / Auto-Fix)

개발자가 에러 로그를 읽고 고치는 과정을 AI가 대신 수행합니다. "실행 → 실패 → 원인 분석 → 수정 → 재실행"의 루프를 자동화했습니다.

### ⚙️ 내부 동작 원리
1. **에러 감지**: 터미널 실행 결과에서 `Error`, `Exception`, `Failed` 등의 키워드와 스택 트레이스(Stack Trace)를 감지합니다.
2. **원인 진단**: 감지된 에러 로그와 관련 소스 코드를 LLM에게 전송하여, 에러의 근본 원인을 분석합니다.
3. **수정 및 검증**: AI가 수정한 코드를 적용하거나 올바른 명령어로 다시 실행을 시도합니다. 이 과정은 설정된 횟수만큼 반복됩니다.

{% hint style="info" %}
**고객 가치**

* **개발 흐름 유지**: 에러로 인한 중단 없이 연속적인 작업 가능
* **디버깅 시간 단축**: AI가 대신 문제를 해결해주어 핵심 로직에 집중 가능
{% endhint %}

---

## 7. 에이전트 오케스트레이션 (Agent Orchestration)

복잡하고 추상적인 요청이 들어오면, 이를 수행 가능한 작은 단위의 작업(Task)으로 쪼개고 순서대로 실행하는 **프로젝트 관리자(PM)** 역할을 수행합니다.

### ⚙️ 내부 동작 원리
1. **계획 수립 (Planning)**: "쇼핑몰 로그인 만들어줘"라는 요청을 받으면 -> [DB 설계, API 구현, UI 작성]으로 할 일을 나눕니다.
2. **서브 에이전트 할당**: 각 할 일에 맞는 전문 에이전트(코딩 에이전트, 터미널 에이전트 등)에게 작업을 위임합니다.
3. **컨텍스트 공유**: 앞 단계의 작업 결과(예: 생성된 DB 스키마)를 다음 단계 에이전트에게 전달하여, 논리적으로 연결된 결과물을 만듭니다.

{% hint style="info" %}
**고객 가치**

* **복잡한 작업 해결**: 단순 코딩을 넘어선 프로젝트 단위의 작업 수행 가능
* **체계적인 개발**: 단계별 진행으로 논리적 오류 최소화
{% endhint %}
