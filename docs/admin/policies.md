# ⚖️ 정책 운영 가이드

Admin에서 설정한 정책은 IDE에 어떻게 적용될까요?  
CodePilot은 유연한 운영을 위해 **두 가지 강도**의 정책 방식을 제공합니다.

## 1. 정책 강도 (Enforcement Level)

### 🔒 강제 정책 (Required)
* **특징**: 사용자가 IDE에서 임의로 변경하거나 끌 수 없습니다.
* **용도**: 보안 규정, 필수 컴플라이언스, 금지 명령어 등
* **예시**: "모든 코드는 실행 전 테스트를 통과해야 한다", "GPT-3.5 사용 금지"

### 🔓 권장 정책 (Recommended)
* **특징**: 기본값으로 설정되지만, 사용자가 필요 시 변경할 수 있습니다.
* **용도**: 생산성 도구 설정, 알림 방식, 선호 모델 등
* **예시**: "코드 리뷰 시 한국어 사용 권장", "자동 완성 기능 켜기"

{% hint style="info" %}
**운영 팁**  
도입 초기에는 **권장 정책** 위주로 시작하여 거부감을 줄이고,  
점차 필수적인 보안 항목을 **강제 정책**으로 전환하는 것이 좋습니다.
{% endhint %}

## 2. 단계별 도입 시나리오

### 🟢 1단계: 도입 및 적응 (PoC)
* **목표**: 개발팀이 AI 도구에 익숙해지고 생산성 효과 체감
* **설정**: 대부분 **권장 정책**으로 설정, 보안상 치명적인 것만 차단
* **효과**: 빠른 확산과 긍정적 사용자 경험 확보

### 🟡 2단계: 표준화 및 안정화
* **목표**: 팀 간 품질 편차를 줄이고 운영 표준 수립
* **설정**: 코딩 컨벤션, 테스트 수행 여부를 **강제 정책**으로 변경
* **효과**: 코드 품질 상향 평준화, 리뷰 비용 감소

### 🔴 3단계: 전사 확산 및 통제
* **목표**: 보안 리스크 최소화 및 비용 최적화
* **설정**: 모델 사용 권한 세분화, 비용 한도 설정, 감사 로그 강화
* **효과**: 대규모 조직에서도 안전하고 효율적인 운영 가능
