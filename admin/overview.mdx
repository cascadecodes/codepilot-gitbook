## 제품 한 줄 소개

> **"우리 회사 AI 개발 환경의 관제탑"**

CodePilot Admin은 조직이 AI 개발 도구를 **안전하고 일관되게 운영**하도록 돕는 통합 관리 콘솔입니다. 개발자 개개인이 제각각 사용하는 AI를 **"회사 기준에 맞는 팀 자산"** 으로 전환합니다.

![Admin 대시보드](/images/ADMIN_DASHBOARD.png)

---

## 왜 Admin이 필요한가요?

| 문제점 | Admin의 해결책 |
| :--- | :--- |
| **품질 편차** — 팀마다 다른 모델·프롬프트를 써서 결과물이 제각각 | **표준 정책 배포**로 일관성 확보 |
| **보안 공백** — 개인이 검증되지 않은 외부 도구를 무분별하게 연결 | **허용 도구·명령어 중앙 통제** |
| **비용 관리** — 누가 얼마나 쓰는지 몰라 비용 예측 불가 | **사용량 모니터링**으로 비용 파악 |

<Info>
CodePilot Admin은 **정책 관리 + 운영 모니터링 + 권한 통제**를 한곳에서 해결하여, 기업이 안심하고 AI를 전사 도입할 수 있는 기반을 제공합니다.
</Info>

---

## 누가 사용하나요?

Admin 콘솔은 **관리자 권한을 가진 사용자만** 접속할 수 있습니다. 일반 팀원은 콘솔에 들어오지 않고, IDE에서 조직 API 키로 가입해 정책을 적용받습니다.

| 역할 | 권한 |
|------|------|
| **오너** | 조직 최고 권한. 관리자 지정, 오너 이전 |
| **관리자** | 모든 설정과 사용자 관리 |
| **팀원** | 코드 작성과 AI 기능 사용 (콘솔 접속 불가) |

## 시작하기

<Steps>
  <Step title="로그인 또는 회원가입">
    [https://codepilot.banya.ai](https://codepilot.banya.ai) 에 관리자 계정으로 접속합니다. 온프레미스 환경은 사내에 배포된 자체 주소를 사용합니다.
  </Step>
  <Step title="조직 만들기">
    새 조직을 만들면 자동으로 관리자 권한을 받습니다.
  </Step>
  <Step title="API 키 발급">
    팀원이 IDE에서 조직에 가입할 때 사용할 API 키를 발급합니다.
  </Step>
  <Step title="정책 설정">
    AI 모델, 보안 규칙, 코딩 컨벤션을 등록합니다.
  </Step>
</Steps>

## 팀 기본 vs 프로젝트

설정은 **팀 기본**과 **프로젝트** 두 단계로 나뉩니다.

- **팀 기본** — 조직 전체에 적용되는 공통 설정
- **프로젝트** — 특정 프로젝트에만 적용되는 설정. 없는 항목은 팀 기본을 상속
