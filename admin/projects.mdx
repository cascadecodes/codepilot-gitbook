## 프로젝트란?

프로젝트는 **조직 하위의 독립적인 설정 단위**입니다. 하나의 조직 안에서 여러 프로젝트를 만들고, 각 프로젝트마다 AI 모델·MCP·RAG·스킬·보안 규칙 등을 다르게 설정할 수 있습니다.

```
조직 (Organization)
├── 팀 기본 설정        ← 모든 프로젝트에 기본 적용
├── 프로젝트 A          ← 프론트엔드 팀 전용 설정
├── 프로젝트 B          ← 백엔드 팀 전용 설정
└── 프로젝트 C          ← AI/ML 팀 전용 설정
```

### 팀 기본 vs 프로젝트 설정

| 구분 | 팀 기본 설정 | 프로젝트 설정 |
|------|-------------|-------------|
| 적용 범위 | 조직 전체 | 해당 프로젝트만 |
| 우선순위 | 기본값 | 팀 기본을 덮어씀 |
| 예시 | "전사 공통 린트 규칙" | "프론트엔드 전용 ESLint 규칙" |

<Info>
프로젝트에 설정이 없는 카테고리는 **팀 기본 설정이 자동 상속**됩니다.
프로젝트 설정이 있으면 해당 항목만 덮어씁니다.
</Info>

## 프로젝트 생성

![프로젝트 생성 화면](/images/ADMIN_PROJECT_CREATE.png)

사이드바 하단의 **프로젝트 선택 드롭다운**에서 프로젝트를 관리합니다.

### 생성 방법

1. 사이드바의 프로젝트 드롭다운에서 **"+ 프로젝트 추가"** 클릭
2. 프로젝트 이름과 설명을 입력
3. **"팀원 모두 추가"** 체크박스 — 조직의 전체 팀원을 자동으로 프로젝트에 추가
4. **생성** 버튼 클릭

<Tip>
**"팀원 모두 추가"** 옵션을 체크하면 현재 조직에 속한 모든 멤버가 자동으로 프로젝트에 추가됩니다. 나중에 개별적으로 추가/제거할 수도 있습니다.
</Tip>

## 프로젝트 전환

![프로젝트 전환 화면](/images/ADMIN_PROJECT_SWITCH.png)

사이드바 상단의 드롭다운에서 프로젝트를 전환합니다.

| 선택 | 효과 |
|------|------|
| **팀 기본** | 조직 전체 공통 설정을 관리 |
| **프로젝트 이름** | 해당 프로젝트의 독립 설정을 관리 |

프로젝트를 선택하면 모든 설정 페이지(AI 모델, MCP, RAG, 스킬, 보안 등)가 **해당 프로젝트 전용 설정**을 표시합니다.

---

## 프로젝트별 설정 관리

프로젝트 모드에서는 각 설정 페이지에 **팀 기본 설정** 패널이 읽기 전용으로 함께 표시됩니다.

![프로젝트 설정 화면](/images/ADMIN_PROJECT_SETTINGS.png)

### 모든 카테고리 지원

프로젝트별로 독립 설정이 가능한 카테고리:

| 카테고리 | 프로젝트 모드에서 |
|---------|----------------|
| AI 모델 | 프로젝트 전용 커스텀 모델 추가 가능, 팀 기본 모델은 읽기 전용 |
| MCP 서버 | 프로젝트 전용 MCP 서버 등록 |
| RAG | 프로젝트 전용 RAG 소스 관리 |
| 스킬/규칙 | 프로젝트 전용 코딩 컨벤션 등록 |
| 빌드/테스트 | 프로젝트 전용 검증 명령어 설정 |
| Hot Load | 프로젝트 전용 자동 대응 규칙 |
| 보안 규칙 | 프로젝트 전용 차단 명령어 추가 |
| 제외 패턴 | 프로젝트 전용 파일 제외 패턴 |

<Info>
팀 기본 설정 패널은 **참고용으로만 표시**됩니다. 팀 기본 설정을 수정하려면 드롭다운에서 "팀 기본"으로 전환하세요.
</Info>

## 프로젝트 멤버 관리

![프로젝트 멤버 관리 화면](/images/ADMIN_PROJECT_MEMBERS.png)

프로젝트 모드에서 **팀 관리** 페이지에 들어가면 해당 프로젝트의 멤버를 관리할 수 있습니다.

### 멤버 추가

1. **팀 관리** 페이지에서 **"멤버 추가"** 버튼 클릭
2. 조직에 속한 팀원 목록에서 검색
3. 추가할 팀원을 선택하여 프로젝트에 추가

### 멤버 제거

- 프로젝트 멤버 목록에서 **탈퇴** 버튼 클릭
- 본인도 프로젝트에서 탈퇴 가능

<Warning>
프로젝트에서 제거된 멤버는 해당 프로젝트의 설정을 더 이상 IDE에서 받지 않습니다. 조직 자체에서 제거되는 것은 아닙니다.
</Warning>

## 프로젝트 수정 / 삭제

사이드바 드롭다운에서 프로젝트 옆의 **편집(✏️)** 또는 **삭제(🗑️)** 아이콘을 클릭합니다.

| 작업 | 설명 |
|------|------|
| **편집** | 프로젝트 이름, 설명 변경 |
| **삭제** | 프로젝트와 모든 프로젝트 전용 설정이 삭제됨 |

<Warning>
프로젝트를 삭제하면 해당 프로젝트에 등록된 **모든 설정, RAG 소스, 멤버 연결**이 함께 삭제됩니다. 이 작업은 되돌릴 수 없습니다.
</Warning>

## 대시보드에서 프로젝트 현황 확인

대시보드에서 프로젝트별 설정 현황을 한눈에 확인할 수 있습니다.

- **팀 기본 모드**: 전체 카테고리별 등록 수, 프로젝트 총 수
- **프로젝트 모드**: 선택한 프로젝트의 카테고리별 설정 수

<Tip>
프로젝트를 처음 만들면 설정이 비어 있습니다. 팀 기본 설정이 자동 상속되므로, **프로젝트만의 차별화가 필요한 항목만** 추가하면 됩니다.
</Tip>
