## 프로젝트란?

프로젝트는 **조직 하위의 독립적인 설정 단위**입니다. 하나의 조직 안에서 여러 프로젝트를 만들고, 각 프로젝트마다 AI 모델·MCP·RAG·규칙·보안 설정을 다르게 적용할 수 있습니다.

```
조직
├── 팀 기본 설정        ← 모든 프로젝트에 기본 적용
├── 프로젝트 A          ← 프론트엔드 팀 전용 설정
├── 프로젝트 B          ← 백엔드 팀 전용 설정
└── 프로젝트 C          ← AI/ML 팀 전용 설정
```

### 팀 기본 vs 프로젝트 설정

| 구분 | 팀 기본 설정 | 프로젝트 설정 |
|------|-------------|-------------|
| 적용 범위 | 조직 전체 | 해당 프로젝트만 |
| 우선순위 | 기본값 | 팀 기본을 덮어씀 |
| 예시 | 전사 공통 린트 규칙 | 프론트엔드 전용 ESLint 규칙 |

<Info>
프로젝트에 설정이 없는 카테고리는 **팀 기본 설정이 자동 상속**됩니다. 프로젝트 설정이 있으면 해당 항목만 덮어씁니다.
</Info>

## 프로젝트 생성

![프로젝트 생성](/images/ADMIN_PROJECT_CREATE.png)

사이드바 위쪽의 **범위 선택 드롭다운**에서 프로젝트를 관리합니다.

<Steps>
  <Step title="프로젝트 추가 클릭">
    드롭다운을 열고 맨 아래 **프로젝트 추가** 를 클릭합니다.
  </Step>
  <Step title="정보 입력">
    프로젝트명과 설명을 입력합니다.
  </Step>
  <Step title="멤버 추가 여부 선택">
    **팀원 전체를 멤버로 추가** 를 체크하면 조직의 모든 멤버가 자동으로 포함됩니다. 나중에 개별 조정할 수 있습니다.
  </Step>
  <Step title="추가">
    생성하면 해당 프로젝트로 자동 전환됩니다.
  </Step>
</Steps>

## 프로젝트 전환

![프로젝트 전환](/images/ADMIN_PROJECT_SWITCH.png)

| 선택 | 효과 |
|------|------|
| **팀 기본** | 조직 전체 공통 설정을 관리 |
| **프로젝트 이름** | 해당 프로젝트의 독립 설정을 관리 |

프로젝트를 선택하면 모든 설정 페이지가 **해당 프로젝트 전용 설정**을 표시합니다.

### 사이드바 메뉴 변화

프로젝트 범위에서는 조직 단위 메뉴가 숨겨집니다.

<CardGroup cols={2}>
  <Card title="팀 기본">
    ![팀 기본 사이드바](/images/ADMIN_SIDEBAR_TEAM.png)
  </Card>
  <Card title="프로젝트">
    ![프로젝트 사이드바](/images/ADMIN_SIDEBAR_PROJECT.png)
  </Card>
</CardGroup>

| 메뉴 | 팀 기본 | 프로젝트 |
|------|:------:|:-------:|
| 대시보드 | O | O |
| AI 모델 · MCP · RAG · Skills | O | O |
| 빌드/테스트 · Hot Load · Hooks · Agents | O | O |
| 보안 규칙 · 제외 패턴 | O | O |
| 팀 관리 | O | O (멤버 관리로 전환) |
| 사용량 모니터링 | O | X |
| API 키 관리 | O | X |
| 접속 로그 · 에러 로그 | O | X |

<Info>
사용량 모니터링, API 키, 로그는 **조직 전체** 단위 기능이므로 프로젝트 범위에서는 표시되지 않습니다.
</Info>

## 프로젝트별 설정 관리

프로젝트 범위에서는 각 설정 페이지 위에 **팀 기본 설정** 패널이 읽기 전용으로 함께 표시됩니다. 상속받은 설정과 프로젝트 전용 설정을 한눈에 비교할 수 있습니다.

![프로젝트 설정](/images/ADMIN_PROJECT_SETTINGS.png)

<Info>
팀 기본 설정을 수정하려면 드롭다운에서 **팀 기본** 으로 전환하세요.
</Info>

<Tip>
프로젝트를 처음 만들면 설정이 비어 있습니다. 팀 기본 설정이 자동 상속되므로, **프로젝트만의 차별화가 필요한 항목만** 추가하면 됩니다.
</Tip>

## 프로젝트 멤버 관리

![프로젝트 멤버 관리](/images/ADMIN_PROJECT_MEMBERS.png)

프로젝트 범위에서 **팀 관리** 메뉴는 **멤버 관리** 로 바뀝니다.

- **멤버 추가** — 조직 멤버 목록에서 검색해 프로젝트에 추가
- **멤버 제거** — 프로젝트에서만 제외 (조직에서 빠지는 것은 아님)
- **프로젝트 탈퇴** — 본인이 직접 프로젝트에서 나가기

<Warning>
프로젝트에서 제거된 멤버는 해당 프로젝트의 설정을 더 이상 IDE에서 받지 않습니다. 조직 자체에서 제거되는 것은 아닙니다.
</Warning>

## 프로젝트 수정 / 삭제

드롭다운에서 프로젝트 위에 마우스를 올리면 편집·삭제 아이콘이 나타납니다.

| 작업 | 설명 |
|------|------|
| **편집** | 프로젝트 이름, 설명 변경 |
| **삭제** | 프로젝트와 모든 프로젝트 전용 설정 삭제 |

<Warning>
프로젝트를 삭제하면 해당 프로젝트의 **모든 설정, RAG 소스, 멤버 연결**이 함께 삭제됩니다. 되돌릴 수 없습니다. 확인을 위해 프로젝트 이름을 정확히 입력해야 합니다.
</Warning>
