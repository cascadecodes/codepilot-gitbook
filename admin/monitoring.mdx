## 대시보드

![Admin 대시보드](/images/ADMIN_DASHBOARD.png)

조직의 상태를 한눈에 확인합니다.

| 영역 | 내용 |
|------|------|
| **통계 카드** | 전체 사용자, 프로젝트 수, 오늘 활성 사용자, API 호출 수, 토큰 사용량, 최근 에러 |

## 사용량 모니터링

![사용량 모니터링](/images/ADMIN_MONITORING.png)

토큰 사용량과 요청 현황을 추적해 비용을 예측합니다.

| 항목 | 설명 |
|------|------|
| 총 토큰 사용량 | 기간 내 전체 토큰 |
| 평균 요청 크기 | 요청당 평균 토큰 |
| 총 요청 수 | AI 호출 횟수 |
| 활성 사용자 | 실제 사용한 인원 |

세부적으로는 **일별 사용량**, **사용자별 사용량**, **모델별 토큰 상세**를 확인할 수 있습니다.

기간은 최근 7일 · 30일 · 90일 또는 직접 지정하고, **CSV 내보내기** 로 데이터를 받아 사내 비용 정산에 활용합니다.

<Tip>
막연한 "감"이 아닌 **정확한 데이터**를 기반으로 운영 비용을 예측하고 최적화합니다. 특정 사용자의 사용량이 과도하면 팀 관리에서 **토큰 한도**를 지정할 수 있습니다.
</Tip>

## 접속 로그

사용자 로그인과 주요 관리 활동 이력을 조회합니다.

{/* SCREENSHOT: 접속 로그 페이지 — 요약 카드(총 활동/성공/실패/최근 활동)와 로그 테이블(시간/사용자/활동/IP/상태) */}
![접속 로그](/images/ADMIN_ACCESS_LOG.png)

기록되는 활동에는 로그인·로그아웃, 조직 생성, 멤버 초대·역할 변경·제거, 설정 추가·수정·삭제, 문서 업로드, API 키 발급 등이 포함됩니다.

**검색·필터**로 특정 사용자나 IP, 성공·실패 여부, 기간을 좁혀 조회하고 CSV로 내보낼 수 있습니다.

<Info>
보안 감사나 컴플라이언스 대응 시, 누가 언제 어떤 설정을 바꿨는지 추적할 수 있습니다.
</Info>

## 에러 로그

시스템 에러와 경고를 수집해 조회합니다.

{/* SCREENSHOT: 에러 로그 페이지 — 레벨별 요약 카드(에러/경고/정보)와 로그 테이블, 행 확장 시 스택 트레이스 */}
![에러 로그](/images/ADMIN_ERROR_LOG.png)

| 필터 | 값 |
|------|-----|
| 레벨 | 심각 · 에러 · 경고 · 정보 |
| 발생 위치 | IDE, AI 응답, 도구 실행, MCP 서버, 인증, 설정 동기화, 파일 입출력, 관리자 웹, 백엔드 |

각 행을 펼치면 발생 시간, 사용자, 상세 내용을 확인할 수 있습니다.

<Tip>
팀원이 "AI가 응답을 안 한다"고 문의하면, 에러 로그에서 해당 사용자와 시간대를 필터링해 원인을 빠르게 확인하세요.
</Tip>
