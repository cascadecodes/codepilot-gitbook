CodePilot IDE는 목적에 따라 네 가지 채팅 모드를 제공합니다. 입력창 왼쪽 드롭다운에서 언제든지 전환할 수 있습니다.

![모드 선택 드롭다운](/images/IDE_MODE_DROPDOWN.png)

| 모드 | 파일 변경 | 명령 실행 | 한 줄 요약 |
|------|:--------:|:--------:|-----------|
| **CODE** | O | O | 지시한 작업을 바로 수행 |
| **AGENT** | O | O | 목표만 주면 스스로 나눠서 수행 |
| **ASK** | X | X | 묻고 답하기 |
| **PLAN** | X | X | 실행 전 계획서 작성 |

---

## CODE 모드

![CODE 모드](/images/IDE_CODE.png)

**파일을 직접 생성·수정·삭제하고 터미널 명령을 실행하는 기본 모드입니다.** 작업 후 빌드·테스트 검증까지 자동으로 이어집니다.

### 언제 사용하나요?

- 새 기능 구현, 버그 수정, 리팩터링
- 프로젝트 초기 설정 (패키지 설치, 설정 파일 생성)
- 테스트 코드 작성
- 반복적인 코드 패턴 일괄 수정

### 예시

```
src/hooks/ 디렉토리에 useAuth 커스텀 훅을 만들어줘.
JWT 토큰 저장, 로그인/로그아웃, 자동 갱신 기능을 포함해줘.
```

```
src/api/ 디렉토리 전체를 검토하고, 에러 처리가 빠진 API 호출에 try-catch를 추가해줘.
```

<Warning>
실제 파일이 변경됩니다. 인라인 Diff에서 Keep / Undo로 변경을 제어할 수 있지만, 중요한 작업 전에는 Git 커밋 상태를 확인하세요.
</Warning>

## AGENT 모드

![AGENT 모드](/images/IDE_AGENT.png)

**목표를 주면 필요한 작업을 스스로 나누고, 필요할 때 보조 에이전트를 만들어 병렬로 처리합니다.**

CODE 모드가 "시킨 일을 정해진 절차대로 한다"면, AGENT 모드는 "목표를 달성할 때까지 알아서 판단한다"에 가깝습니다.

| | CODE | AGENT |
|---|------|-------|
| 작업 순서 | 정해진 절차를 따름 | AI가 스스로 판단 |
| 빌드 검증 | 자동 수행 | 필요하다고 판단할 때 수행 |
| 속도 | 안정적, 검증 시간 추가 | 빠름, 모델 품질에 더 의존 |

### 언제 사용하나요?

- 여러 파일·여러 계층에 걸친 큰 작업
- 무엇부터 손대야 할지 스스로 판단이 필요한 작업
- 구현 후 테스트·검증까지 한 번에 맡기고 싶을 때

### 예시

```
결제 모듈에 환불 기능을 추가하고, 관련 테스트까지 작성한 뒤 전체 테스트를 돌려줘.
```

진행 상황은 채팅 패널의 작업 목록에서 실시간으로 확인할 수 있습니다.

![작업 진행 상황](/images/IDE_JOB_QUEUE.png)

<Tip>
**프론티어 모델을 사용한다면 AGENT 모드를 권장합니다.**

최신 고성능 모델은 작업 순서와 검증 시점을 스스로 판단하는 능력이 뛰어납니다. 정해진 절차를 따르는 CODE 모드보다, 모델의 판단에 맡기는 AGENT 모드에서 더 좋은 결과를 냅니다.
</Tip>

<Warning>
AGENT 모드는 AI의 자율 판단에 의존합니다. 경량 모델이나 로컬 모델을 사용 중이고 안정적이고 예측 가능한 결과가 필요하다면 CODE 모드를 사용하세요.
</Warning>

## ASK 모드

![ASK 모드](/images/IDE_ASK.png)

**파일을 수정하지 않고 질문하고 답변을 받는 모드입니다.**

### 할 수 있는 것

- 코드 설명 및 분석
- 설계 방향 논의
- 오류 원인 분석
- 코드 리뷰 의견 요청

### 언제 사용하나요?

- 낯선 코드베이스를 처음 파악할 때
- 특정 함수나 클래스의 동작을 이해하고 싶을 때
- 코드 변경 없이 리뷰 의견만 받고 싶을 때

### 예시

```
@src/auth/session.ts 이 파일이 하는 역할을 설명해줘. 주요 함수와 데이터 흐름 위주로 알려줘.
```

```
현재 인증 구조에서 보안상 취약한 부분이 있으면 알려줘. 수정은 하지 말고 분석만 해줘.
```

## PLAN 모드

![PLAN 모드](/images/IDE_PLAN.png)

**파일을 수정하지 않고 구현 계획을 먼저 수립하는 모드입니다.** 코드베이스를 읽어 분석한 뒤, 변경 대상 파일·구현 단계·예상 리스크를 정리한 계획서를 출력합니다.

### 언제 사용하나요?

- 큰 기능 구현 전에 영향 범위를 미리 파악할 때
- 여러 파일에 걸친 변경이 예상될 때
- 팀원과 계획을 공유하고 검토받은 뒤 실행하고 싶을 때

### 예시

```
결제 시스템을 추가하려고 해. Stripe 연동, 결제 내역 저장, 환불 처리까지 어떻게 구현하면 될까?
```

### 계획 검토 후 실행하기

PLAN 모드의 핵심은 **계획을 검토한 뒤 실행**하는 흐름입니다.

<Steps>
  <Step title="PLAN 모드로 계획 수립">
    원하는 기능을 요청하면 AI가 코드베이스를 탐색하고 계획서를 출력합니다.
  </Step>
  <Step title="계획 검토 및 조정">
    변경 대상 파일과 구현 단계를 확인합니다. 필요하면 추가 질문으로 계획을 다듬습니다.

    ```
    3단계에서 기존 라우터를 수정하지 말고 새 라우터를 만드는 게 낫지 않을까?
    ```
  </Step>
  <Step title="CODE 모드로 전환해 실행">
    모드를 CODE로 바꾸고 실행을 요청합니다. 이전 대화에 계획이 남아 있어 그대로 이어집니다.

    ```
    위 계획대로 구현해줘.
    ```
  </Step>
</Steps>

### 사용 중인 모델에 따라

| 사용 모델 | 권장 모드 | 이유 |
|----------|----------|------|
| **프론티어 모델** (최신 고성능 상용 모델) | **AGENT** | 작업 분해와 검증 시점을 스스로 잘 판단합니다 |
| 경량 모델 · 로컬 모델 | **CODE** | 정해진 절차를 따르므로 결과가 안정적입니다 |

기본 모드는 설정 → **일반** → **기본 모드** 에서 바꿀 수 있습니다.
</Tip>
