IDE 내 설정 화면은 사용자와 관리자 정책이 만나는 접점입니다.

![설정 화면 예시](/images/IDE_SETTING.png)

## 주요 설정 항목

### 계정 (Account)

![계정 설정 화면](/images/IDE_ACCOUNT.png)

* 로그인 상태 및 조직 연동 정보를 확인합니다.
* 조직에 소속되면 **관리자 정책(Admin Policy)**이 자동으로 내려옵니다.

<Tip>
**설명 포인트**
* 개인 계정과 조직 계정을 명확히 구분하여 보안 정책 적용 여부를 시각적으로 확인합니다.
</Tip>

### 일반 설정 (General)

![일반 설정 화면](/images/IDE_GENERAL.png)

* 언어, 테마, 알림 등 기본적인 사용 환경을 설정합니다.
* 자동 실행 허용 여부 등 개인 선호를 반영할 수 있습니다.
* **소스코드 자동완성**: Ghost Text(인라인 Tab 완성) 기능을 ON/OFF로 제어합니다. 기본값은 OFF이며, 활성화 시 타이핑 중 AI가 코드를 실시간으로 제안합니다.

<Tip>
**설명 포인트**
* 개발자의 작업 스타일에 맞춰 알림 빈도나 자동화 수준을 세밀하게 조정할 수 있습니다.
* 소스코드 자동완성은 전용 경량 모델을 라우팅 모델 설정에서 지정할수록 응답 속도가 빨라집니다.
</Tip>

### AI 모델 (AI Model)

![AI 모델 설정 화면](/images/IDE_AI_MODEL.png)

* **조직 허용 모델**: 회사가 비용/보안을 고려해 지정한 모델만 사용 가능
* **개인 모델**: 필요 시 개인 API Key를 등록하여 사용 (정책 허용 시)
* 상황에 따라 성능과 속도 중심 모델을 선택하여 사용합니다.
* **라우팅 모델 슬롯**: 의도 분석 / 명령 실행 / 컨텍스트 압축 / 에러 폴백 / **소스코드 자동완성** 용도별로 독립 모델을 지정할 수 있습니다. 소스코드 자동완성에는 `starcoder2:3b`, `deepseek-coder:1.3b`, `qwen2.5-coder:1.5b-base` 같은 경량 로컬 모델을 권장합니다.

<Tip>
**설명 포인트**
* 비용 효율적인 모델과 고성능 모델을 상황에 맞게 선택하여 운영 비용을 최적화합니다.
* 소스코드 자동완성 전용 슬롯에 Ollama 로컬 모델을 지정하면 API 비용 없이 빠른 완성이 가능합니다.
</Tip>

### 도구 연동 (Tools & MCP)

![MCP 도구 설정 화면](/images/IDE_MCP.png)

* 외부 도구(Jira, GitHub, DB 등)를 IDE와 연결합니다.
* **MCP(Model Context Protocol)**를 통해 확장 가능한 도구 생태계를 지원합니다.

<Tip>
**설명 포인트**
* IDE를 벗어나지 않고도 외부 시스템과 연동하여 업무 흐름을 끊지 않고 작업할 수 있습니다.
</Tip>

### 지식 검색 (RAG)

![RAG 설정 화면](/images/IDE_RAG.png)

* 사내 문서 저장소를 연결하여 검색 범위를 설정합니다.
* 프로젝트별로 참조할 문서를 다르게 지정할 수 있습니다.

<Tip>
**설명 포인트**
* 우리 회사만의 도메인 지식을 AI가 학습하여, 일반적인 답변이 아닌 맞춤형 코드를 제안합니다.
</Tip>

### 빌드/테스트 (Build & Test)

![빌드/테스트 설정 화면](/images/IDE_BUILD.png)

* 코드 생성 후 자동으로 실행할 **검증 명령어**를 설정합니다.
* 예: `npm test`, `eslint`, `mvn verify` 등
* 품질 기준을 통과하지 못하면 AI가 스스로 수정을 시도합니다.

<Tip>
**설명 포인트**
* 코드를 작성하는 즉시 검증하여, 버그가 포함된 코드가 커밋되는 것을 원천 차단합니다.
</Tip>

### 자동 대응 (Hot Load)

![핫로드 설정 화면](/images/IDE_HOTLOAD.png)

* 특정 상황에서 반복되는 작업을 자동으로 수행하도록 설정합니다.
* 실패 시 재시도 또는 다른 처리 방식도 선택할 수 있습니다.

<Tip>
**설명 포인트**
* 자주 발생하는 에러나 반복 작업을 자동화 규칙으로 등록하여 개발 피로도를 줄입니다.
</Tip>

### 스킬 및 규칙 (Skills)

![스킬 설정 화면](/images/IDE_SKILLS.png)

* 팀의 코딩 가이드와 아키텍처 원칙을 일관되게 적용합니다.

<Tip>
**설명 포인트**
* 신규 입사자도 별도의 교육 없이 팀의 코딩 스타일을 준수하며 개발할 수 있습니다.
</Tip>

### 보안 (Security)

![보안 설정 화면](/images/IDE_SCRURITY.png)

* **차단 목록**: 실행을 막을 명령어 패턴 등록
* **제외 목록**: AI가 읽지 말아야 할 파일/폴더 지정 (예: `.env`, `secrets/`)

<Tip>
**설명 포인트**
* 실수로 인한 시스템 파괴나 민감 정보 유출을 시스템 레벨에서 안전하게 방어합니다.
</Tip>
