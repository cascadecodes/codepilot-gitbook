## 계정

![계정 설정](/images/IDE_ACCOUNT.png)

로그인 상태와 소속 조직을 확인하고, 조직 가입과 프로젝트 선택을 처리하는 화면입니다.

| 항목 | 설명 |
|------|------|
| 프로필 | 로그인 계정, 소속 조직, 인증 상태 |
| 조직 연결 | 조직 API 키(`cpk_...`) 등록 및 탈퇴 |
| 프로젝트 | 작업할 프로젝트 선택 |
| 조직 설정 동기화 | 서버에서 최신 정책 다시 받아오기 |

자세한 절차는 [조직 연결](/getting-started/organization) 문서를 참고하세요.

### 언어 / 테마 / 기본 모드

| 항목 | 설명 |
|------|------|
| 언어 | AI 응답 언어 (한국어 / 영어) |
| 테마 | 다크 / 라이트 / VS Code 설정 따르기 |
| 기본 모드 | 채팅 시작 시 선택될 모드 |

### 자동 실행 권한

AI가 파일을 수정하거나 명령을 실행하기 전에 **확인을 요청할지, 바로 실행할지**를 정합니다.

| 설정 | ON일 때 | OFF일 때 |
|------|---------|---------|
| 도구 자동 실행 | AI가 도구를 바로 실행 | 실행 전 승인 요청 |
| 파일 자동 업데이트 | 수정 사항을 즉시 적용 | Diff 확인 후 승인 |
| 명령 자동 실행 | 터미널 명령을 바로 실행 | 실행 전 확인 |

<Warning>
처음 사용한다면 **도구 자동 실행과 명령 자동 실행은 OFF** 로 두고 시작하세요. AI가 어떤 작업을 하는지 확인하며 신뢰를 쌓은 뒤 켜는 것을 권장합니다.
</Warning>

### 그 외

- **소스코드 자동추천** — 타이핑 중 Tab 완성 (기본값 OFF)
- **다음 작업 제안** — 작업 완료 후 후속 작업 제안
- **웹검색** — 답변에 웹 검색 결과 반영
- **자격 증명 보호** — 응답에 비밀 키가 노출되지 않도록 검사
- **설정 내보내기 / 가져오기** — 설정을 JSON 파일로 백업·복원

## AI 모델

![AI 모델 설정](/images/IDE_AI_MODEL.png)

### 메인 모델 선택

채팅에 사용할 기본 모델을 고릅니다.

| 구분 | 특징 |
|------|------|
| 클라우드 모델 | 고성능, 인터넷 필요, 비용 발생 |
| 로컬 모델 | 무료, 오프라인 가능, 속도는 PC 사양에 따라 다름 |

<Tip>
조직에 연결된 경우 관리자가 승인한 모델만 목록에 표시됩니다.
</Tip>

### 응답 옵션

| 옵션 | 설명 |
|------|------|
| 스트리밍 | 답변을 생성하는 대로 표시 |
| Thinking | 더 깊이 추론 (Low / Medium / High / Max) |
| 멀티모달 | 이미지·PDF 첨부 지원 |

### API 키 등록

| 환경 | 방법 |
|------|------|
| 조직에 로그인 | **API Key** 항목에서 **관리자 API Key**(조직 부담) 또는 **개인 API Key**(본인 부담) 중 선택 |
| 독립 실행(BYOK) | **사용자 정의 모델** 에서 제공사를 골라 모델과 API 키를 직접 등록 |

<Card title="AI 모델 연결하기" icon="key" href="/getting-started/ai-model">
  Gemini API 키를 발급받아 등록하는 방법을 단계별로 안내합니다
</Card>

### 용도별 모델 지정

작업 종류마다 다른 모델을 지정할 수 있습니다. 간단한 작업에 가벼운 모델을 배정하면 비용과 응답 속도를 함께 개선할 수 있습니다.

| 용도 | 권장 |
|------|------|
| 대화 압축 | 경량 모델 |
| 명령 실행 | 메인 모델 |
| 의도 분석 | 경량 모델 |
| 에러 대비 | 메인 모델과 다른 모델 |
| 소스코드 자동추천 | 가벼운 로컬 모델 |

## MCP 서버

![MCP 설정](/images/IDE_MCP.png)

GitHub, Jira, 사내 시스템 등 외부 도구를 AI가 직접 다룰 수 있도록 연결합니다.

- **로컬 연결** — 내 PC에서 실행되는 도구
- **원격 연결** — 외부 MCP 서버 URL

인증은 API 키 또는 OAuth를 지원합니다.

<Info>
조직 관리자가 MCP 서버를 미리 등록해두면, 별도 설정 없이 바로 사용할 수 있습니다.
</Info>

## RAG (사내 문서)

![RAG 설정](/images/IDE_RAG.png)

관리자가 Admin 콘솔에 등록한 사내 문서 목록을 확인합니다. 채팅 시 관련 문서를 자동으로 검색해 답변에 반영합니다.

문서 등록과 인덱싱은 관리자가 수행하므로, IDE에서는 어떤 문서가 연결돼 있는지만 확인합니다.

## 빌드/테스트

![빌드/테스트 설정](/images/IDE_BUILD.png)

코드를 만들거나 고친 뒤 자동으로 실행할 검증 명령을 지정합니다.

| 프로젝트 | 명령어 예시 |
|---------|-----------|
| Node.js | `npm test`, `npm run lint`, `npm run build` |
| Python | `pytest` |
| Java (Maven) | `mvn verify` |
| Java (Gradle) | `./gradlew test` |

비워두면 프로젝트 유형을 감지해 자동으로 선택합니다. 검증에 실패하면 AI가 원인을 분석해 수정한 뒤 다시 검증합니다.

## Hot Load

![Hot Load 설정](/images/IDE_HOTLOAD.png)

자주 쓰는 작업을 키워드로 등록하면, 채팅에 그 키워드가 담긴 문장을 입력할 때 등록된 명령이 실행됩니다.

```
키워드:  배포, deploy
설명:    프로덕션 배포 실행
명령어:  ./scripts/deploy.sh production
```

완료 조건(종료 코드, 출력 문자열, 파일 존재 여부)과 재시도 횟수, 실패 시 동작을 함께 지정할 수 있습니다.

## Skills (스킬 및 규칙)

![스킬 설정](/images/IDE_SKILLS.png)

AI가 코드를 작성할 때 따라야 할 팀 규칙을 Markdown으로 등록합니다.

```
모든 React 컴포넌트는 함수형으로 작성하고,
Props는 interface로 타입을 정의한다.
```

카테고리는 전역 규칙, 버전 관리, 코딩 스타일, 프로젝트 아키텍처, 의존성 정책, DB로 나뉩니다.

| 종류 | 설명 |
|------|------|
| 로컬 규칙 | 내 PC에서만 적용 |
| 조직 규칙 | 관리자가 등록해 모든 팀원에게 적용 |

<Info>
조직 규칙 중 **필수** 항목은 개인이 수정하거나 끌 수 없습니다.
</Info>

## 보안

![보안 설정](/images/IDE_SCRURITY.png)

AI가 실행하면 안 되는 명령과 접근하면 안 되는 파일을 지정합니다.

### 차단 명령어

시스템을 파괴하거나 되돌릴 수 없는 명령을 막습니다.

```
rm -rf /
DROP TABLE
git push --force
chmod 777
```

### 보호 파일 / 제외 패턴

AI가 읽거나 수정하지 못하도록 막을 경로를 지정합니다.

```
.env              ← API 키, 비밀번호
.env.local
secrets/          ← 인증서, 비밀 키
config/secrets.yml
```

<Info>
`.gitignore`에 등록된 경로는 파일 탐색 시 자동으로 건너뜁니다. 다만 완전히 차단하려면 제외 패턴에 직접 추가하세요.
</Info>

<Warning>
보안 설정은 AI가 실수로 중요한 파일을 건드리는 것을 막는 마지막 안전장치입니다. `.env` 파일은 반드시 제외 패턴에 추가하세요.
</Warning>

## 프로젝트별 설정

프로젝트를 선택하면 설정 화면에 **팀 기본 설정**이 읽기 전용으로 함께 표시됩니다. 프로젝트에 별도 설정이 없는 항목은 팀 기본 설정을 그대로 따릅니다.

![프로젝트 보안 설정](/images/IDE_PROJECT_SECURITY.png)

<AccordionGroup>
  <Accordion title="프로젝트를 선택하지 않으면 어떻게 되나요?">
    팀 기본 설정이 적용됩니다. 관리자가 등록한 조직 공통 설정이 그대로 사용됩니다.
  </Accordion>
  <Accordion title="프로젝트 목록이 비어 있어요">
    관리자가 아직 프로젝트를 만들지 않았거나, 해당 프로젝트의 멤버로 추가되지 않은 경우입니다. 관리자에게 문의하세요.
  </Accordion>
  <Accordion title="프로젝트를 바꾸면 대화 내용이 사라지나요?">
    아닙니다. 대화는 유지되고, 다음 메시지부터 새 프로젝트의 설정이 적용됩니다.
  </Accordion>
  <Accordion title="로컬 규칙과 조직 규칙이 충돌하면?">
    조직 규칙이 우선합니다. 로컬 규칙은 조직 규칙을 보완하는 용도로 사용하세요.
  </Accordion>
</AccordionGroup>
