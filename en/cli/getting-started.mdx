The CodePilot CLI brings the **same AI engine** as the IDE to your terminal. It shares organization settings and sign-in state with the IDE, so configuring one applies to the other.

<Info>
If you work in VS Code, the [IDE extension](/en/getting-started/install) alone is all you need. Use the CLI for terminal-centric workflows or when integrating with CI pipelines.
</Info>

---

## Installation

Get the executable from your organization administrator. No separate runtime installation is required.

<Steps>
  <Step title="Get the executable">
    Obtain the CodePilot CLI executable for your operating system and unzip it.
  </Step>
  <Step title="Add it to your PATH">
    Move the executable to a directory that is included in your PATH.

    ```bash
    # macOS / Linux example
    chmod +x codepilot
    sudo mv codepilot /usr/local/bin/
    ```
  </Step>
  <Step title="Verify the installation">
    ```bash
    codepilot --version
    ```
  </Step>
</Steps>

## Running

Run it from the project directory you want to work in.

```bash
cd ~/projects/my-app
codepilot
```

On first launch, a start screen appears showing your current model, mode, folder, and account details.

{/* SCREENSHOT: Welcome screen shown on first CLI launch — logo box with the Model/Mode/Permission/Folder/User/Organization/Project list */}
![CLI start screen](/images/CLI_WELCOME.png)

<Info>
The first time you run it in a folder, you are asked whether to trust it. In folders you have not trusted, the AI will not modify files or run commands.
</Info>

## Sign in

If you launch without being signed in, the sign-in screen appears first. If the CLI is already running, type `/login`.

{/* SCREENSHOT: CLI sign-in gate screen — "🔒 Sign-in required" message with Enter to start sign-in */}
![CLI sign-in](/images/CLI_LOGIN.png)

```bash
/login              # Sign in with the default provider
/login github       # Sign in with GitHub
/login microsoft    # Sign in with Microsoft
```

A browser opens to complete authentication. Once you are done, return to the terminal to continue working.

<Tip>
In on-premises environments, instead of a browser you are prompted to enter your email and password directly in the terminal.
</Tip>

### Check sign-in status

```bash
/whoami
```

Shows your account, organization, role, and the API key in use.

## Connect to an organization

Join an organization using the organization API key (`cpk_...`) you received from your administrator. You must be signed in first.

```bash
/join cpk_xxxxxxxxxxxx
```

Once you join, the organization's AI model policies, security rules, and coding conventions are applied.

```bash
/project      # Select a project
/sync         # Re-fetch the latest organization settings
/join leave   # Leave the organization
```

## Your first conversation

Type your request directly at the prompt.

```
Explain the structure of this project.
```

| Input | Action |
|------|------|
| Plain text | Sends the request to the AI |
| Starts with `/` | Runs a slash command |
| Starts with `@` | Attaches a file or an agent |
| Starts with `!` | Runs a shell command directly, without the AI |

### Key shortcuts

| Shortcut | Action |
|--------|------|
| `Enter` | Send message |
| `Alt+Enter` | New line |
| `Shift+Tab` | Switch permission mode (Auto → Edit → Ask) |
| `Ctrl+T` | Toggle the Thinking level |
| `Esc` | Cancel input / stop generation |
| `Ctrl+C` | Exit |

---

## Permission modes

Controls how often the AI asks before running a tool. Switch with `Shift+Tab`.

| Mode | Behavior |
|------|------|
| **Auto** | Automatically approves every tool execution |
| **Edit** | Auto-approves file edits; asks before running or deleting |
| **Ask** | Asks before every tool execution |

<Warning>
Until you are comfortable, start in **Ask** or **Edit** mode.
</Warning>
