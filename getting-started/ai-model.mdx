CodePilot이 답변을 만들려면 **AI 모델**이 연결되어 있어야 합니다. 연결 방법은 사용 환경에 따라 다릅니다.

| 상황 | 해야 할 일 |
|------|-----------|
| 조직에 연결되어 있다 | 관리자가 이미 모델을 등록해 두었습니다. [모델을 고르기만](#조직에-연결된-경우) 하면 됩니다 |
| 개인/독립 실행으로 쓴다 | [API 키를 직접 발급받아 등록](#1-gemini-api-키-발급받기)합니다 |

이 문서에서는 **Google Gemini**를 예로 들어 처음부터 끝까지 설명합니다. 다른 제공사도 절차는 같습니다.

---

## 1. Gemini API 키 발급받기

API 키는 Google에서 무료로 발급받을 수 있습니다.

<Steps>
  <Step title="Google AI Studio 접속">
    브라우저에서 [aistudio.google.com/apikey](https://aistudio.google.com/apikey) 에 접속하고 Google 계정으로 로그인합니다.
  </Step>
  <Step title="API 키 만들기">
    **API 키 만들기(Create API key)** 버튼을 클릭합니다. 프로젝트를 고르라고 하면 아무 프로젝트나 선택하거나 새로 만듭니다.
  </Step>
  <Step title="키 복사">
    `AIza` 로 시작하는 문자열이 생성됩니다. **복사** 버튼을 눌러 보관하세요.
  </Step>
</Steps>

<Warning>
API 키는 비밀번호와 같습니다. 소스 코드나 사내 메신저에 붙여넣지 마세요. 유출되면 Google AI Studio에서 즉시 삭제하고 새로 발급받으세요.
</Warning>

<Info>
Gemini는 무료 사용량이 제공됩니다. 사용량을 초과하면 과금될 수 있으니, Google AI Studio에서 결제 설정과 한도를 확인하세요.
</Info>

## 2. CodePilot에 모델 등록하기

<Info>
이 화면은 **로그인 없이 사용하는 독립 실행(BYOK) 환경**에서만 나타납니다. 조직에 로그인한 상태라면 [조직에 연결된 경우](#조직에-연결된-경우)로 넘어가세요.
</Info>

<Steps>
  <Step title="설정 열기">
    채팅 패널 오른쪽 위의 **톱니바퀴(⚙) 아이콘**을 클릭합니다.
  </Step>

  <Step title="AI 모델 탭 열기">
    왼쪽 메뉴에서 **AI 모델** 을 선택합니다.

    ![AI 모델 설정](/images/IDE_AI_MODEL.png)
  </Step>

  <Step title="모델 추가">
    **사용자 정의 모델** 항목에서 **+ 모델 추가** 를 클릭합니다.

    {/* SCREENSHOT: 설정 > AI 모델 탭의 "사용자 정의 모델" 영역 — "+ 모델 추가" 버튼과 빈 목록 안내 문구 */}
    ![모델 추가 버튼](/images/IDE_MODEL_ADD.png)
  </Step>

  <Step title="프로바이더 선택">
    **프로바이더** 드롭다운에서 **Google Gemini** 를 고릅니다.

    선택하면 **API 엔드포인트**가 자동으로 채워집니다. 이 값은 그대로 두세요.

    {/* SCREENSHOT: "새 AI 모델 추가" 폼 — 프로바이더가 Google Gemini로 선택되고 API 엔드포인트가 자동 입력된 상태 */}
    ![Gemini 모델 등록 폼](/images/IDE_MODEL_FORM_GEMINI.png)
  </Step>

  <Step title="모델 정보 입력">
    아래 두 항목을 직접 입력합니다.

    | 항목 | 입력할 값 |
    |------|----------|
    | **모델 이름** | `Gemini 3.1 Pro` — 목록에 표시될 이름이므로 자유롭게 |
    | **모델 ID** | `gemini-3.1-pro-preview` — Google이 정한 정확한 값이어야 합니다 |

    <Warning>
    입력창에 흐리게 표시된 예시(`gemini-2.5-pro`)는 예전 모델입니다. 그대로 쓰지 말고 위 값을 직접 입력하세요.
    </Warning>
  </Step>

  <Step title="API 키 붙여넣기">
    **API 키** 항목에 1단계에서 복사한 키를 붙여넣습니다.

    <Info>
    라벨에 "(선택)"이라고 쓰여 있지만, Gemini를 쓰려면 **반드시 입력해야 합니다.** 이 항목이 선택인 이유는, 인증이 필요 없는 사내 모델 서버도 등록할 수 있기 때문입니다.
    </Info>
  </Step>

  <Step title="저장">
    나머지 항목(인증 방식, 컨텍스트 윈도우, 온도, 추가 헤더 등)은 **손대지 말고 기본값 그대로 두세요.** 비워두면 모델의 기본값이 적용됩니다.

    **저장** 을 클릭하면 목록에 모델이 추가됩니다.
  </Step>
</Steps>

## 3. 연결 확인하고 사용 시작하기

저장한 모델 옆의 버튼으로 확인과 활성화를 진행합니다.

{/* SCREENSHOT: 사용자 정의 모델 목록에서 등록된 Gemini 항목 — 테스트/선택/편집/삭제 버튼과 "API 키 저장됨" 표시 */}
![모델 테스트와 선택](/images/IDE_MODEL_TEST.png)

<Steps>
  <Step title="테스트">
    **테스트** 를 클릭합니다. **연결 정상** 이라고 나오면 성공입니다.

    **연결 실패** 가 나오면 아래 [문제 해결](#문제-해결)을 확인하세요.
  </Step>
  <Step title="선택">
    **선택** 을 클릭하면 **✓ 활성** 표시가 붙고, 이 모델이 실제 채팅에 사용됩니다.

    <Warning>
    저장만 하고 **선택**하지 않으면 모델이 등록만 될 뿐 사용되지 않습니다. 반드시 선택까지 하세요.
    </Warning>
  </Step>
  <Step title="채팅에서 확인">
    채팅 패널로 돌아가면 모델 선택 드롭다운의 **사용자 정의** 항목에 방금 등록한 모델이 보입니다. 여기서도 바로 전환할 수 있습니다.
  </Step>
</Steps>

이제 채팅에 아무 질문이나 입력해 보세요.

```
안녕? 잘 연결됐는지 확인용이야.
```

## 조직에 연결된 경우

조직에 로그인해 사용하는 경우에는 **사용자 정의 모델** 화면이 나타나지 않습니다. 관리자가 이미 조직에서 쓸 모델을 등록해 두었기 때문입니다.

설정 → **AI 모델** 에서 모델을 고르고, **API Key** 항목에서 어떤 키를 쓸지 선택합니다.

{/* SCREENSHOT: 조직 로그인 상태의 설정 > AI 모델 — "API Key" 항목에 관리자 API Key / 개인 API Key 선택 드롭다운 */}
![API 키 소스 선택](/images/IDE_MODEL_KEY_SOURCE.png)

| 선택 | 의미 |
|------|------|
| **관리자 API Key** | 조직이 등록한 키를 씁니다. 비용은 조직이 부담합니다 |
| **개인 API Key** | 내가 발급받은 키를 씁니다. 비용은 내가 부담합니다 |

개인 키를 쓰려면 **개인 API Key** 입력창에 키를 붙여넣고 **저장** 을 클릭합니다. 위 [1단계](#1-gemini-api-키-발급받기)에서 발급받은 Gemini 키를 그대로 쓸 수 있습니다.

<Info>
등록된 키가 하나도 없으면 **등록된 키 없음** 으로 표시됩니다. 이 경우 관리자에게 문의하거나 개인 키를 등록하세요.
</Info>

## 다른 제공사 사용하기

프로바이더 드롭다운에서 다른 제공사를 고르면 됩니다. 절차는 Gemini와 동일하고, **API 키를 발급받는 곳**만 다릅니다.

| 제공사 | 모델 ID 예시 |
|--------|------------|
| OpenAI | `gpt-4o` |
| Anthropic Claude | `claude-opus-4-6` |
| Google Gemini | `gemini-3.1-pro-preview` |
| DeepSeek | `deepseek-chat` |

<Info>
모델 ID는 제공사가 새 모델을 내놓을 때마다 바뀝니다. 위 값이 동작하지 않으면 각 제공사의 공식 문서에서 최신 모델 ID를 확인하세요.
</Info>

클라우드(Vertex AI · Azure OpenAI · AWS Bedrock)와 사내 서버(OpenAI 호환 엔드포인트)도 등록할 수 있습니다.

<Tip>
인터넷 없이 쓰고 싶다면 API 키가 필요 없는 **로컬 모델**을 쓸 수 있습니다. 설정 → AI 모델에서 로컬 모델 서버를 연결하세요.
</Tip>

## 문제 해결

<AccordionGroup>
  <Accordion title="테스트에서 '연결 실패'가 나옵니다">
    다음 순서로 확인하세요.

    1. **API 키** — 복사할 때 앞뒤 공백이 섞이지 않았는지 확인합니다. 다시 붙여넣어 보세요.
    2. **모델 ID** — `gemini-3.1-pro-preview` 처럼 Google이 정한 정확한 값이어야 합니다. 모델 이름(`Gemini 3.1 Pro`)과 혼동하지 마세요. 입력창의 회색 예시를 그대로 두지 않았는지도 확인하세요.
    3. **API 엔드포인트** — 프로바이더를 다시 선택하면 올바른 값으로 되돌아옵니다.
    4. **네트워크** — 사내 방화벽이 외부 API 호출을 막고 있을 수 있습니다. 관리자에게 문의하세요.
  </Accordion>

  <Accordion title="'사용자 정의 모델' 항목이 안 보입니다">
    조직에 로그인한 상태입니다. 이 경우 관리자가 등록한 모델을 사용하며, 직접 모델을 추가할 수 없습니다. [조직에 연결된 경우](#조직에-연결된-경우)를 참고하세요.
  </Accordion>

  <Accordion title="모델을 저장했는데 채팅에서 답변이 안 옵니다">
    저장만으로는 적용되지 않습니다. 모델 목록에서 **선택** 버튼을 눌러 **✓ 활성** 상태로 만들었는지 확인하세요.
  </Accordion>

  <Accordion title="API 키를 다시 입력해야 하나요?">
    한 번 저장하면 안전하게 보관됩니다. 모델을 수정할 때 API 키 항목을 비워두면 기존 키가 그대로 유지됩니다.
  </Accordion>

  <Accordion title="할당량 초과 오류가 납니다">
    Gemini 무료 사용량을 다 썼을 수 있습니다. Google AI Studio에서 사용량과 결제 설정을 확인하세요.
  </Accordion>
</AccordionGroup>
