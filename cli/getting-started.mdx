CodePilot CLI는 IDE와 **같은 AI 엔진**을 터미널에서 사용하는 도구입니다. 조직 설정과 로그인 정보를 IDE와 공유하므로, 한쪽에서 설정하면 다른 쪽에도 적용됩니다.

<Info>
VS Code에서 작업한다면 [IDE 확장](/getting-started/install)만으로 충분합니다. CLI는 터미널 중심 작업이나 CI 파이프라인 연동이 필요할 때 사용하세요.
</Info>

---

## 설치

조직 관리자에게 실행 파일을 전달받습니다. 별도의 런타임 설치는 필요 없습니다.

<Steps>
  <Step title="실행 파일 받기">
    운영체제에 맞는 CodePilot CLI 실행 파일을 전달받아 압축을 풉니다.
  </Step>
  <Step title="PATH에 등록">
    실행 파일을 PATH에 포함된 디렉토리로 옮깁니다.

    ```bash
    # macOS / Linux 예시
    chmod +x codepilot
    sudo mv codepilot /usr/local/bin/
    ```
  </Step>
  <Step title="설치 확인">
    ```bash
    codepilot --version
    ```
  </Step>
</Steps>

## 실행

작업할 프로젝트 디렉토리에서 실행합니다.

```bash
cd ~/projects/my-app
codepilot
```

처음 실행하면 시작 화면이 나타나고, 현재 모델·모드·폴더·계정 정보를 보여줍니다.

{/* SCREENSHOT: CLI 첫 실행 시 나타나는 환영 화면 — 로고 박스와 Model/Mode/Permission/Folder/User/Organization/Project 목록 */}
![CLI 시작 화면](/images/CLI_WELCOME.png)

<Info>
처음 실행하는 폴더는 신뢰 여부를 묻습니다. 신뢰하지 않은 폴더에서는 AI가 파일을 수정하거나 명령을 실행하지 않습니다.
</Info>

## 로그인

로그인하지 않은 상태로 실행하면 로그인 화면이 먼저 나타납니다. 이미 실행 중이라면 `/login` 을 입력합니다.

{/* SCREENSHOT: CLI 로그인 게이트 화면 — "🔒 로그인이 필요합니다" 안내와 Enter로 로그인 시작 */}
![CLI 로그인](/images/CLI_LOGIN.png)

```bash
/login              # 기본 제공자로 로그인
/login github       # GitHub으로 로그인
/login microsoft    # Microsoft로 로그인
```

브라우저가 열리며 인증이 진행됩니다. 인증을 마치면 터미널로 돌아와 작업을 계속합니다.

<Tip>
온프레미스 환경에서는 브라우저 대신 터미널에서 이메일·비밀번호를 입력하는 화면이 나타납니다.
</Tip>

### 로그인 상태 확인

```bash
/whoami
```

계정, 소속 조직, 역할, 사용 중인 API 키를 확인합니다.

## 조직 연결

관리자에게 받은 조직 API 키(`cpk_...`)로 조직에 가입합니다. 로그인 후에 실행해야 합니다.

```bash
/join cpk_xxxxxxxxxxxx
```

가입하면 조직의 AI 모델 정책, 보안 규칙, 코딩 컨벤션이 적용됩니다.

```bash
/project      # 프로젝트 선택
/sync         # 최신 조직 설정 다시 받아오기
/join leave   # 조직 탈퇴
```

## 첫 대화

프롬프트에 요청을 그대로 입력합니다.

```
이 프로젝트의 구조를 설명해줘.
```

| 입력 | 동작 |
|------|------|
| 일반 문장 | AI에게 요청 전송 |
| `/` 로 시작 | 슬래시 명령어 실행 |
| `@` 로 시작 | 파일이나 에이전트 첨부 |
| `!` 로 시작 | AI 없이 셸 명령 바로 실행 |

### 주요 단축키

| 단축키 | 동작 |
|--------|------|
| `Enter` | 메시지 전송 |
| `Alt+Enter` | 줄바꿈 |
| `Shift+Tab` | 권한 모드 전환 (Auto → Edit → Ask) |
| `Ctrl+T` | Thinking 단계 전환 |
| `Esc` | 입력 취소 / 생성 중단 |
| `Ctrl+C` | 종료 |

## 권한 모드

AI가 도구를 실행하기 전에 얼마나 자주 물어볼지 정합니다. `Shift+Tab` 으로 전환합니다.

| 모드 | 동작 |
|------|------|
| **Auto** | 모든 도구 실행을 자동 승인 |
| **Edit** | 파일 수정은 자동, 명령 실행·삭제는 확인 |
| **Ask** | 모든 도구 실행 전에 확인 |

<Warning>
익숙해지기 전까지는 **Ask** 또는 **Edit** 모드로 시작하세요.
</Warning>
