## 모드 전환

IDE와 동일하게 네 가지 모드를 지원합니다.

```bash
/mode           # 선택 창 열기
/mode agent     # 바로 전환
```

| 모드 | 파일 변경 | 용도 |
|------|:--------:|------|
| `code` | O | 지시한 작업을 바로 수행 |
| `agent` | O | 목표만 주면 스스로 나눠서 수행 (기본값) |
| `ask` | X | 질문과 분석 |
| `plan` | X | 실행 전 계획서 작성 |

시작 모드는 실행 시 지정할 수도 있습니다.

```bash
codepilot --mode plan
```

## 슬래시 명령어

`/` 를 입력하면 명령어 목록이 자동완성으로 나타납니다. `Tab` 으로 선택하고 `Enter` 로 실행합니다.

{/* SCREENSHOT: CLI 입력창에서 "/" 입력 시 뜨는 슬래시 명령어 자동완성 목록 */}
![슬래시 명령어 자동완성](/images/CLI_SLASH.png)

### 계정 · 조직

| 명령어 | 설명 |
|--------|------|
| `/login [provider]` | 로그인 |
| `/logout` | 로그아웃 |
| `/whoami` | 계정·조직·API 키 확인 |
| `/join <cpk_...>` | 조직 가입 |
| `/project` | 프로젝트 선택 |
| `/sync` | 조직 설정 다시 받아오기 |
| `/trust` | 현재 폴더 신뢰 설정 |

### 모델 · 동작

| 명령어 | 설명 |
|--------|------|
| `/model` | 모델 목록·변경 |
| `/thinking` | 추론 단계 변경 (off / low / medium / high) |
| `/web on\|off` | 웹 검색 사용 여부 |
| `/orchestration on\|off` | 멀티 에이전트 사용 여부 |
| `/settings` | 설정 확인 및 변경 |

### 대화 관리

| 명령어 | 설명 |
|--------|------|
| `/clear` | 대화 초기화 |
| `/resume` | 이전 세션 복원 |
| `/session` | 세션 저장·목록·삭제·내보내기 |
| `/compact` | 대화 압축 (토큰 절약) |
| `/status` | 이번 세션의 사용량 통계 |
| `/memory` | 장기 기억 관리 |
| `/init` | 프로젝트를 분석해 `AGENTS.md` 작성 |

### Git · 개발 워크플로우

| 명령어 | 설명 |
|--------|------|
| `/diff` | 커밋되지 않은 변경 사항 |
| `/pending` | 승인 대기 중인 파일 변경 |
| `/accept [파일]` | 변경 사항 적용 |
| `/undo [파일]` | 변경 사항 되돌리기 |
| `/commit "메시지"` | 커밋 (`/commit ai` 로 메시지 자동 생성) |
| `/commit-push-pr` | 커밋 → 푸시 → PR 생성 |
| `/review [기준브랜치]` | 현재 브랜치 변경분 리뷰 |
| `/pr-comments` | PR 리뷰 코멘트 반영 |
| `/autofix-pr` | CI 실패 로그 분석 후 자동 수정 |
| `/security-review` | 보안 취약점 검토 |

### 도구 · 규칙

| 명령어 | 설명 |
|--------|------|
| `/mcp` | MCP 서버 관리 |
| `/agents` | 서브에이전트 관리 |
| `/skills` | 스킬·규칙 관리 |
| `/rules` | 조직 규칙 확인·토글 |
| `/rag` | 사내 문서 소스 관리 |
| `/hooks` | 훅 관리 |
| `/hotload` | 키워드 자동 실행 규칙 |
| `/buildtest` | 빌드·테스트 명령 설정 |
| `/security` | 보안 규칙 관리 |

<Tip>

## 파일 첨부와 셸 실행

```bash
# 파일을 컨텍스트에 첨부
@src/api/auth.ts 토큰 만료 처리를 개선해줘

# 서브에이전트 호출
@security-reviewer 방금 추가한 인증 코드를 검토해줘

# AI를 거치지 않고 셸 명령 바로 실행
!git status
```

`!` 로 실행한 명령의 출력은 다음 메시지의 컨텍스트에 자동으로 포함됩니다.

이미지나 PDF 파일 경로를 붙여넣으면 첨부로 인식합니다.

## 자동화 실행 (비대화형)

`-p` 옵션을 주면 대화형 화면 없이 한 번만 실행하고 결과를 출력한 뒤 종료합니다. CI 파이프라인이나 스크립트에서 사용합니다.

```bash
codepilot -p "현재 테스트 실패 원인을 요약해줘"
```

{/* SCREENSHOT: 터미널에서 codepilot -p "..." 실행 결과가 표준 출력으로 나온 화면 */}
![비대화형 실행](/images/CLI_HEADLESS.png)

### 주요 옵션

| 옵션 | 설명 |
|------|------|
| `-p, --print <프롬프트>` | 한 번 실행하고 종료 |
| `--json` | 결과를 JSON 형식으로 출력 |
| `--mode <모드>` | 시작 모드 지정 |
| `--image <경로>` | 이미지·PDF 첨부 |
| `--permission-mode accept\|reject` | 도구 실행 승인 정책 |
| `--yes` | 신뢰하지 않은 폴더에서도 실행 승인 |
| `--no-banner` | 시작 배너 생략 |
| `--version` | 버전 확인 |

### 파이프로 연결하기

응답은 표준 출력으로, 진행 상황은 표준 오류로 나가므로 그대로 파이프할 수 있습니다.

```bash
codepilot -p "이 커밋의 변경 요약을 한 문장으로" > summary.txt
codepilot -p "테스트 실패 원인" --json | jq '.text'
```

<Warning>
비대화형 실행에서는 도구 실행 확인이 자동으로 승인됩니다. CI에서 사용할 때는 `--permission-mode reject` 로 파일 변경을 막거나, 격리된 환경에서 실행하세요.
</Warning>

### CI에서 사용하기

미리 발급받은 토큰을 환경 변수로 넘기면 로그인 없이 실행됩니다.

```bash
export CODEPILOT_ACCESS_TOKEN="..."
export CODEPILOT_BACKEND_URL="https://codepilot.mycompany.com/api/v1"

codepilot -p "변경된 코드의 보안 취약점을 검토해줘" --yes --json
```

<Warning>
토큰은 CI 시크릿으로 관리하세요. 저장소나 로그에 노출되지 않도록 주의해야 합니다.
</Warning>

## 프로젝트 컨텍스트 파일

프로젝트 루트에 **`AGENTS.md`** 파일이 있으면 대화 시작 시 자동으로 컨텍스트에 포함됩니다. 프로젝트 개요, 아키텍처, 코딩 규칙을 적어두면 매번 설명할 필요가 없습니다.

```bash
/init    # 프로젝트를 분석해 AGENTS.md 초안 생성
```
